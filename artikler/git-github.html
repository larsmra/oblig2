Git / Github

3. oktober 2018

Hva er git?

Git er et versjonskontrollsystem[1]. Det ble utviklet av Linus Torvalds i 2005, og er i dag det mest brukte versjonskontrollsystemet i verden[1][2]. Et versjonskontrollsystem er et system som tar vare på eldre versjoner av et prosjekt. Dette gjør det enkelt for utviklere å se endringer i prosjektet, og gå tilbake til tidligere versjoner av prosjektet i tilfelle den nye koden ikke fungerer som den skal[3].

Når en person i en gruppe vil gjøre en endring på et prosjekt, blir det laget en kopi av den delen av koden som skal endres som personen kan arbeide med uten å gjøre endringer i hovedkoden. Når koden fungerer som den skal, kan den slås sammen med resten av koden[3].

Github

GitHub er et selskap som tilbyr lagringsplass for git-prosjekter[3]. Prosjekter som bruker git, lagres i noe som kalles git-repositories[1]. Github-brukere kan opprette åpne git-repositories på Github sine servere[3]. Github brukes hovedsakelig av programutviklere, men det er ingenting i veien for å bruke GitHub som en vanlig skylagringstjeneste hvor man oppbevarer versjoner av vanlige tekstdokumenter eller bilder[1].

I denne oppgaven har jeg lastet opp filene for denne obligen i en git-repository. Repositoriet kan du finne ved å klikke her. Personlig mener jeg at Git er et bra værktøy for utviklere, spesielt hvis man er en stor gruppe som jobber med et stort prosjekt, da det ellers hadde vært vanskelig å holde styr på alle oppdateringene som blir gjort.
